Task 6.2: Cart Item Card

Horizontal cart item card component with product image, name, category, price, quantity controls, and remove action. No borders - spacing-based separation.

APPROVED

Approved Design Choices

  • Mobile (375px): Option A (Compact Stacked) + trash icon positioned absolute top-right (from Option C)
  • Tablet (768px): Option A (Compact Stacked) — trash icon inline next to qty selector
  • Desktop (1440px): Option A (Table Layout) — grid with Product/Price/Quantity/Total columns

CURRENT BEHAVIOR

  • Basic card layout
  • Inconsistent sizing
  • No swipe-to-delete on mobile
  • Small touch targets

NEW DESIGN

  • Horizontal layout: Image (80px) + info
  • Name, category, price, quantity in info area
  • Compact connected pill quantity selector
  • Subtle trash icon for remove
  • No borders, soft shadow, rounded-2xl
  • Larger touch targets on mobile (36px min)

Notes

Mobile uses Option A (Compact Stacked) layout but adds a trash/delete icon positioned absolute in the top-right corner (borrowed from Option C) so users always have a visible remove affordance on mobile.

Implementation Specs (Task 6.2):

• Image: 80px mobile, 88px tablet, 100-120px desktop
• Layout: Horizontal flex, gap-16px
• Name: font-display, font-semibold, truncate
• Category: text-muted, text-sm
• Price: font-display, font-bold, text-primary
• Quantity: Connected pill selector (Task 4.5 component)
• Remove: Trash icon, muted, hover:red
• Card: rounded-2xl, shadow-soft, no borders
