Task 8.3: Wishlist Page

Wishlist page matching products page layout pattern. Content-integrated header, sort toolbar, product grid, compact list variant, and empty state.

Awaiting Approval

CURRENT BEHAVIOR

  • Basic wishlist grid
  • Inconsistent page header
  • No stock status indicators
  • No empty state design
  • Inconsistent card styles across viewports

NEW DESIGN

  • Listing-page layout matching products page (content-integrated header, sort toolbar)
  • Product cards with stock status (In Stock / Low / Out of Stock)
  • Sort toolbar with item count and sort dropdown
  • Remove button and Add to Cart on each card
  • Disabled state for out-of-stock items
  • Engaging empty state with CTA
  • Responsive grid: 2-col mobile, 3-col tablet, 4-col desktop

Notes / Feedback
Implementation Specs (Task 8.3):

• Layout Pattern: Matches products page (ListingPageHeader style) - content-integrated header, sort toolbar, responsive grid
• Page Header: Breadcrumbs (Home > Wishlist), title, subtitle - no green underline, no separate white bar
• Mobile Header: Back button + centered title (h-14, 56px)
• Sort Toolbar: Item count on left, sort dropdown on right (Recently Added, Price Low-High, Price High-Low)
• Grid Layout: Mobile 2-col, Tablet 3-col, Desktop 4-col
• Card Corners: rounded-2xl (16px)
• Shadows: shadow-soft default, shadow-card on hover
• Image Aspect: 1:1 (square)
• Hover Effect: translateY(-2px) + shadow increase
• Stock Status Colors: In Stock (#4F6F52), Low Stock (#C4785A), Out of Stock (#EF4444)
• Out of Stock: Disable add-to-cart, show "Notify Me"
• Empty State: Primary-colored circle icon (#E8ECE8 bg, #4F6F52 icon), centered layout
• Files: src/features/wishlist/components/wishlist.tsx
