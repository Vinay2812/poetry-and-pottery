Mobile (375px) - Key Decisions

Everything stacks in a single column for a natural thumb-scrolling experience. The flow is strictly linear: config, dates, slots, participants, pricing.

  • Tabs: 3-tab navigation with "Daily Workshops" as active tab with green underline. Compact font size (12px) to fit all three tabs.
  • Config cards: Full-width stacked cards with radio selection. Selected card has green border + light green background. Tier info shown as inline chips.
  • Calendar: Week view is default on mobile to save vertical space. Week/Month toggle available. Past dates are dimmed/disabled. Today gets a green dot.
  • Time slots: 2-column grid (3 would be too cramped at 375px). Each slot is 12px rounded card with centered time and availability text.
  • Slot states: Available = white with subtle green border on hover. Selected = solid #4F6F52 fill, white text. Full = gray background, 60% opacity, "Full" label. Blocked = diagonal stripe pattern, line-through time, "Blocked" label.
  • Multi-date: Selected dates appear as chips that can be tapped to scroll to that date's slots. Active date chip is filled green.
  • Pricing: Full-width panel at bottom. Shows optimal tier grouping with per-person price. Savings callout in green background. "Proceed to Payment" is a full-width rounded-full primary button.
  • Step numbers: Green numbered circles (1-4) create a visual flow and help orientation during scroll.

Tablet (768px) - Key Decisions

At 768px we gain enough room for side-by-side config cards and 3-column time slots, but the overall flow remains vertical (no sidebar pricing yet).

  • Config cards: 2-column grid. Both configs visible at a glance without scrolling. Radio + label stays inline.
  • Calendar: Month view is default on tablet (enough space). Full calendar grid with green dots for dates that have available slots. Selected dates filled green.
  • Time slots: 3-column grid (vs. 2 on mobile). All 6 slots per date visible in 2 rows. More scannable layout.
  • Multiple dates: Shows 3 dates selected (Feb 6, 7, 13) to demonstrate cross-week booking. Each date's slot grid has its own header.
  • Tab styling: Standard 13px font tabs with comfortable padding. Green underline on active.
  • Pricing panel: Still full-width below the slots. Shows more complex pricing (5 slots grouped into 2×2hr + 1×1hr tiers).
  • Selection chips: Wrap naturally across multiple lines. Each has an × button for removal.
  • Scroll depth: Longer than mobile because month view + 3 dates' worth of slots are shown. Acceptable for tablet scrolling patterns.

Desktop (1200px) - Key Decisions

The main layout shift at desktop: pricing moves to a sticky sidebar on the right (340px). The main content column gets the remaining space for config, calendar, and slots. This keeps pricing always visible while the user selects slots.

  • Layout: 2-column grid: main content + 340px sticky sidebar. Sidebar holds the pricing panel, submit button, and cancellation policy note.
  • Time slots: 6-column grid on desktop (all 6 slots in a single row per date). Maximum scannability: one glance to see an entire day's availability.
  • Config cards: Side-by-side at full width. More room for longer descriptions and all tier chips.
  • Calendar: Month view default. Comfortably sized cells. Green dots indicate dates with available slots. Multiple dates can be selected (green fill).
  • Pricing sidebar: Sticky positioning means the breakdown stays visible as the user scrolls through date selection and slot picking. Live updates as selections change.
  • Smart pricing callout: Moved inside the sidebar for constant visibility. Shows tier optimization explanation.
  • Participants: Includes "Max 6 per slot" subtitle since there is space. Counter controls are the same across viewports.
  • Savings indicator: Green pill inside pricing panel shows how much the user saves vs. individual 1-hr bookings. Encouraging upsell.

Component Reference: Smart Pricing Logic
The system groups consecutive selected slots into the best possible tier combination. Below shows how different slot selections map to tier combos.
Slots Selected Optimal Tiers Per Person Pieces Savings
1 PM (single slot) 1 hr tier ₹950 1 pc —
1 PM + 2 PM (consecutive) 2 hr tier ₹1,700 2 pcs ₹200
1 PM + 2 PM + 3 PM (consecutive) 3 hr tier ₹2,400 3 pcs ₹450
1 PM + 2 PM + 4 PM (gap at 3 PM) 2 hr + 1 hr ₹2,650 3 pcs ₹200
1-3 PM (Day A) + 2 PM (Day B) 3 hr + 1 hr ₹3,350 4 pcs ₹450
Implementation Notes:
Slot Hours: Fixed 6 slots per day: 1:00 PM, 2:00 PM, 3:00 PM, 4:00 PM, 5:00 PM, 6:00 PM IST (each 1 hour).
Multi-date: Users can select dates across different weeks. Each date shows its own slot grid. Date chips act as anchors to scroll to that date's slots.
Tier Optimization: The frontend groups consecutive slots on the same day into the longest possible tier first (greedy algorithm). Non-consecutive slots or cross-day slots remain individual tiers.
Capacity: Each slot has a max_capacity (e.g. 6). Current bookings are tracked. Remaining = max - booked. When remaining = 0, slot shows "Full".
Admin Blackout: Admin can mark specific date+slot combos as blocked. These show with the striped pattern and are never selectable.
Responsive Grid: Mobile: 2-col slots. Tablet: 3-col slots. Desktop: 6-col slots (all slots in one row).
Calendar Default: Mobile shows week view (compact). Tablet/Desktop show month view. User can toggle.
Pricing Sidebar: Desktop only. On mobile/tablet the pricing panel is at the bottom of the flow. On desktop it is sticky in a 340px right sidebar.
Selected State Persistence: Selections are held in client state (Zustand). Navigating between tabs preserves selections. Changing config type clears all slot selections.
Accessibility: All slots have aria-labels. Keyboard navigation supported. Focus indicators visible. Screen reader announces slot state.
