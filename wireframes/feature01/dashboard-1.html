Desktop (1200px)

Full desktop layout with persistent sidebar navigation and wide main content area. All 4 tabs are previewed vertically in sequence to show the complete admin interface. The "Daily Workshops" nav item is highlighted as active, positioned after "Events" in the sidebar.

  • ✓ Persistent sidebar with "Daily Workshops" active (calendar+clock icon)
  • ✓ Tab 1 - Config: Full table with all columns (ID, Name, Description, Status, Timezone, Hours, Slot, Capacity, Booking Window)
  • ✓ Tab 2 - Pricing Tiers: Config selector dropdown, tier table with hours/price/pieces/active/sort
  • ✓ Tab 3 - Blackout Rules: Type badges (ONE_TIME, WEEKLY, MONTHLY), "Preview Impact" modal
  • ✓ Tab 4 - Registrations: Filter bar (status, date range, search), status badges, actions dropdown
  • ✓ Toggle switches for Active/Inactive states on configs, tiers, and rules
Implementation Notes:
Route: /dashboard/daily-workshops
Sidebar Nav: "Daily Workshops" item with calendar+clock icon, positioned after "Events" in the navigation order.
Tab 1 - Config: CRUD for workshop configurations. Toggle switch for Active/Inactive status. Fields: name, description, status, timezone, operating hours, slot duration, max capacity, booking window.
Tab 2 - Pricing Tiers: Config-scoped pricing management. Config selector dropdown at top. Fields: hours, price_per_person, pieces_per_person, is_active, sort_order.
Tab 3 - Blackout Rules: Availability disabling rules. Types: ONE_TIME (specific dates), WEEKLY (recurring day), MONTHLY (recurring date). "Preview Impact" modal shows affected registrations count.
Tab 4 - Registrations: All daily workshop bookings with filter bar. Actions: Approve, Reject, Mark Paid, Cancel (with reason input). Status flow: Pending -> Approved -> Paid -> Confirmed | Cancelled.
Responsive: Mobile uses card lists + horizontal tab scroll. Tablet uses condensed tables. Desktop uses sidebar + full tables.

Mobile / Tablet Behavior
On mobile and tablet viewports, the Kanban board collapses into a filterable list view. Users can filter by status using a dropdown selector, and registration cards are displayed as stacked cards with status badges. Drag-and-drop is not available on smaller screens; status changes use an actions dropdown instead.

User Detail — DW Registrations Kanban (Desktop)

The user detail view at /dashboard/users/[id] now includes a fifth tab for Daily Workshop Registrations. When selected, it displays a Kanban board with 6 status columns matching the DailyWorkshopRegistrationStatus enum. Admins can drag cards between columns to update registration status. This follows the same pattern already used for Orders and Event Registrations tabs.

  • ✓ 5 tab cards in grid: Orders, Event Registrations, DW Registrations (new), Cart Items, Wishlist
  • ✓ 6-column Kanban: Pending (amber), Approved (green), Paid (blue), Confirmed (dark green), Rejected (red), Cancelled (gray)
  • ✓ Each card shows: Registration ID (monospace), workshop config name, date/time slot chips, hours, participants, pieces, and amount
  • ✓ Discount badges appear on cards where applicable (e.g., "10% off")
  • ✓ Drag handle on each card for status column reordering
  • ✓ Hover shadow effect on cards for visual drag affordance
  • ✓ Desktop only: Mobile/tablet collapses to filterable list with status dropdown and action buttons
Implementation Notes (User Detail — DW Registrations Tab):
Route: /dashboard/users/[id] (new tab within existing user detail page)
New Tab: "Daily Workshop Registrations" with calendar+clock icon, added as the 3rd tab between Event Registrations and Cart Items.
Kanban Board: 6 columns matching DailyWorkshopRegistrationStatus enum: Pending, Approved, Paid, Confirmed, Rejected, Cancelled. Each column has a color-coded header with count badge.
Card Data: Registration ID (NanoID prefixed "DW-"), workshop config name, date/time slot chips, total hours, participant count, total pieces, amount with optional discount badge.
Drag & Drop: Cards can be dragged between columns to update status. Drag handle icon (6-dot grip) visible on each card. Backend validates status transitions.
Responsive: Kanban board is desktop-only (1024px+). On mobile/tablet, collapses to a filterable card list with status dropdown filter and action buttons for status changes.
Status Colors: Pending (#f59e0b amber), Approved (#22c55e green), Paid (#3b82f6 blue), Confirmed (#15803d dark green), Rejected (#ef4444 red), Cancelled (#a3a3a3 gray).
