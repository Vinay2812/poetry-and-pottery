Implementation Specs (Option 2 -- Split Panel Layout):

• Layout Pattern: CSS Grid split panel -- `grid-template-columns: 1fr 340px` on desktop, `1fr 260px` on tablet, `1fr` stacked on mobile
• Sticky Sidebar: `position: sticky; top: 16px` on the pricing panel so it follows scroll on desktop/tablet
• Calendar Grid: CSS Grid with `grid-template-columns: 60px repeat(7, 1fr)` for 7-day view; reduce columns for tablet (5) and mobile (3)
• Slot States: 4 visual states -- Available (white + hover:green-border), Selected (#4F6F52 bg + white checkmark), Full (#f5f5f4 + "Full"), Blocked (diagonal stripe + red text)
• Mobile Bottom Sheet: Fixed at viewport bottom, collapsed by default showing slot count + total + book CTA. Expandable via drag handle to show full breakdown
• Tier Optimization: Client-side algorithm groups consecutive selected hours into optimal tier blocks (1hr, 2hr, 3hr) to minimize cost per person
• Participants: +/- selector, min 1, max defined by workshop config (typically 6). Total recalculates: perPerson x participants
• Week Navigation: Left/right arrows to navigate weeks. Today's date column highlighted with green date number
• Responsive Breakpoints: Desktop >= 1024px (7-day + 340px sidebar), Tablet 768-1023px (5-day + 260px sidebar), Mobile < 768px (3-day stacked + bottom sheet)
• Data Flow: Fetch available slots per week via `dailyWorkshopSlots(configId, weekStart)` query. Slot availability updates in real-time via polling or subscription
• Key Advantage vs Other Options: Pricing CTA always visible on desktop/tablet without scrolling. Split panel is a well-understood e-commerce pattern (similar to Airbnb, Calendly)
