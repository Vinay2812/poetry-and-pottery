Task 11.3: Scroll Animations

Staggered reveal animations on product grids and scroll-triggered fade-in effects. Desktop only with reduced motion support. Play each demo below to see and compare the three animation options.

Awaiting Approval

Animation Behavior by Device

📱 Mobile (375px) DISABLED

Scroll animations are disabled on mobile for performance and battery optimization. Elements appear immediately.

Reason: Limited processing power; animations cause jank and drain battery.
📱 Tablet (768px) DISABLED

Scroll animations are disabled on tablet. Touch scrolling with animations can feel sluggish.

Note: Hover/tap feedback animations still work.
🖥️ Desktop (1440px) ENABLED

Scroll animations are enabled on desktop. Staggered reveals create visual delight when scrolling.

Experience: Elements fade in from bottom as they enter viewport.

Reduced Motion Support

All animations respect the user's prefers-reduced-motion system preference. When enabled, elements appear instantly with no animation.

/* CSS fallback for reduced motion */ @media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } } // React hook for Framer Motion const prefersReducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches; // Skip animation when user prefers reduced motion <motion.div initial={prefersReducedMotion ? false : "hidden"} whileInView={"visible"} viewport={{ once: true, amount: 0.2 }} />

Notes / Feedback
Implementation Specs (Task 11.3):

• Library: Framer Motion with whileInView trigger
• Viewport: Trigger when 20% of element is visible
• Once: Animation plays only once (not on scroll back)
• Media Query: Only enable for lg: breakpoint and above
• Reduced Motion: Respect prefers-reduced-motion
• Files: product-card.tsx, event-card.tsx (wrap with motion.div)
